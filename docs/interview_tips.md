# 技术面试技巧

> 算法能力只是一半，表达和沟通同样重要

## 🎯 面试前准备

### 1. 刷题数量
- **最低标准**：100+ 道
- **推荐**：Hot 100 全部完成
- **质量 > 数量**：每道题要真正理解

### 2. 知识体系
确保覆盖以下核心知识点：
- [ ] 数组、字符串、链表
- [ ] 栈、队列、哈希表
- [ ] 二叉树、图
- [ ] 排序、二分查找
- [ ] 双指针、滑动窗口
- [ ] DFS、BFS
- [ ] 动态规划
- [ ] 贪心

### 3. 模拟面试
- 找朋友互相模拟
- 使用在线模拟面试平台
- 录下自己做题过程回看

## 💬 面试中的沟通

### 1. 读题后不要立即写代码

**标准流程：**
1. 复述题目，确认理解正确
2. 确认边界条件和特殊情况
3. 说出你的思路
4. 得到面试官认可后再写代码

**示例对话：**
> "让我确认一下，这道题是要找数组中两个数的和等于 target，返回它们的下标对吗？输入保证有唯一解吗？数组可能是空的吗？"

### 2. 边写边说

不要沉默地写代码，保持沟通：
- "我先写一个哈希表来存储已遍历的元素"
- "这里用双指针，左指针从头，右指针从尾"
- "这个递归的终止条件是..."

### 3. 卡住了怎么办？

1. **说出你的思考过程**："我在想是否可以用哈希表优化..."
2. **请求提示**："您能给我一些提示吗？"
3. **从暴力解开始**："我先说一个暴力解法，然后我们看怎么优化"

## 📊 复杂度分析

**主动说明，不等面试官问：**

> "这个解法的时间复杂度是 O(n)，因为我们只遍历了一次数组。空间复杂度是 O(n)，因为哈希表最多存储 n 个元素。"

**如果有多种解法：**

> "暴力解法是 O(n²)，用哈希表可以优化到 O(n)，空间换时间。"

## ❌ 常见错误

### 1. 边界处理不当
- 空数组/空字符串
- 只有一个元素
- 所有元素相同

### 2. 整数溢出
- Python 不用担心，但用其他语言要注意

### 3. 链表操作
- 空指针访问
- 丢失指针导致链表断开

### 4. 递归问题
- 没有终止条件
- 终止条件不完整
